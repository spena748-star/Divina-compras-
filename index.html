🔧 Firebase

Toca esto solo si cambiaste de proyecto Firebase.

🛡️ Reglas de seguridad recomendadas

Copia estas reglas en Firebase Console → Firestore → Reglas y en Storage → Reglas. Evitan que extraños escriban datos aunque vean tu config.

// ── FIRESTORE ──────────────────────────
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    // Config: solo lectura pública, escritura bloqueada
    match /config/{doc} {
      allow read: if true;
      allow write: if false;
    }
    // Usuarios: lectura pública (para login), escritura bloqueada
    match /usuarios/{uid} {
      allow read: if true;
      allow write: if false;
    }
    // Ventas: lectura pública, escritura bloqueada desde fuera
    match /ventas/{id} {
      allow read: if true;
      allow write: if false;
    }
    // Comprobantes y notificaciones: igual
    match /{collection}/{doc} {
      allow read: if true;
      allow write: if false;
    }
  }
}

// ── STORAGE ────────────────────────────
rules_version = '2';
service firebase.storage {
  match /b/{bucket}/o {
    match /comprobantes/{file} {
      allow read: if true;
      allow write: if false;
    }
  }
}

⚠️ Con estas reglas la app deja de poder escribir desde el navegador. Para producción real activa Firebase App Check en la consola y cambia allow write: if false por allow write: if request.app != null — así solo tu dominio puede escribir.

🔑 Restricción de API Key (muy recomendada):
Ve a Google Cloud Console → Credenciales, selecciona tu API Key de Firebase y en "Restricciones de aplicación" agrega solo tu dominio (ej: tudominio.pages.dev). Así el key no sirve desde otros sitios aunque alguien lo vea.

📖 Manual de captura

Guía rápida para capturar artículos correctamente. También puedes pasar el cursor sobre cada campo del formulario para ver una ayuda rápida.

📋 Flujo del pedido
1

Captura los artículos del cliente

En la pestaña ➕ Agregar, escribe el código de 4 dígitos del cliente y agrega cada artículo. Verás la lista en tiempo real en el panel dorado de abajo.

💡 El código son los últimos 4 dígitos del WhatsApp del cliente.
2

Presiona "Captura terminada"

Cuando termines de agregar todos los artículos, presiona el botón verde. Se abrirá WhatsApp con un resumen listo para enviarle al cliente.

3

El cliente revisa y aprueba

El cliente entra a la app con su código, ve sus artículos y presiona ✅ o ❌ en cada uno según lo que quiera.

💡 Solo los artículos aprobados se cobran.
4

El cliente sube su comprobante

Después de aprobar, el cliente sube la foto o PDF de su transferencia desde su pantalla.

5

Tú verificas y liberas

Ve a la pestaña 👤 Clientes, busca su código, revisa el comprobante y presiona ✅ Liberar pago. Los artículos quedan marcados como Pagados.

📝 Campos del formulario
?

Código cliente

Los últimos 4 dígitos del número de WhatsApp del cliente. Debe estar registrado en 👥 Usuarios. Ejemplo: si su número es 65612345678, el código es 5678.

?

Producto y Marca

Escribe el nombre del artículo tal como aparece en la etiqueta o en la tienda. La marca va separada. Ejemplo: Producto: Tenis running / Marca: Nike.

?

Precio USD

El precio en dólares tal como está en la etiqueta de la tienda, sin impuesto. El sistema convierte automáticamente a MXN usando el tipo de cambio configurado y agrega la comisión.

💡 El total en MXN aparece como vista previa abajo del formulario.
?

Tipo de artículo y Talla

Calzado: selecciona categoría y talla USA — el sistema muestra el equivalente en México y CM.
Ropa: selecciona género y talla (XS–4XL) con equivalente en número mexicano.
Ropa interior: selecciona tipo (brasier, boxer, panty…). Para brasier elige banda + copa (ej: 34B).
Otro: escribe libremente lo que necesites.

?

Banco

La cuenta bancaria donde el cliente debe depositar. Si aún no lo sabes, déjalo en blanco y edítalo después.

?

Vendedor (Shopper)

El vendedor que compró el artículo. Esto se usa para calcular su comisión del 10% cuando el pago sea liberado.

?

Nota

Cualquier observación extra: color, estilo, número de referencia, condición del artículo, etc.

💡 La nota le aparece visible al cliente en su pantalla.
